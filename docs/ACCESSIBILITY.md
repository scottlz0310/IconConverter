## 概要

IconConverterは、すべてのユーザーが利用できるように設計されています。WCAG 2.1 AAレベルの基準に準拠し、様々な支援技術と互換性があります。

### アクセシビリティの目標

- ✅ **知覚可能**: すべての情報とUIコンポーネントが認識可能
- ✅ **操作可能**: すべての機能がキーボードで操作可能
- ✅ **理解可能**: 情報とUIの操作が理解しやすい
- ✅ **堅牢**: 様々な支援技術で利用可能

---

## WCAG 2.1準拠

IconConverterは、Web Content Accessibility Guidelines (WCAG) 2.1のレベルAA基準に準拠しています。

### 準拠状況

| 原則 | レベル | 状態 | 備考 |
|------|--------|------|------|
| 知覚可能 | AA | ✅ 準拠 | 代替テキスト、色のコントラスト |
| 操作可能 | AA | ✅ 準拠 | キーボード操作、フォーカス管理 |
| 理解可能 | AA | ✅ 準拠 | 明確なラベル、エラーメッセージ |
| 堅牢 | AA | ✅ 準拠 | セマンティックHTML、ARIA属性 |

### 主要な準拠項目

#### 1.1 代替テキスト

- すべての画像に適切な代替テキストを提供
- アイコンにはaria-labelを設定

#### 1.4 識別可能

- テキストと背景のコントラスト比: 最低4.5:1
- ダークモード: 最低7:1
- フォーカスインジケーター: 明確に表示

#### 2.1 キーボードアクセス可能

- すべての機能がキーボードで操作可能
- キーボードトラップなし

#### 2.4 ナビゲート可能

- 論理的なタブ順序
- スキップリンクの提供
- 明確なページタイトル

#### 3.1 読みやすさ

- 明確で簡潔な言語
- 専門用語の説明

#### 3.2 予測可能

- 一貫したナビゲーション
- 予期しない動作なし

#### 3.3 入力支援

- 明確なエラーメッセージ
- 入力の検証とフィードバック

#### 4.1 互換性

- セマンティックHTML
- ARIA属性の適切な使用

## キーボードナビゲーション

すべての機能はキーボードのみで操作できます。

### 基本的なキーボードショートカット

#### グローバルショートカット

| キー | 機能 | Windows/Linux | macOS |
|------|------|---------------|-------|
| ファイルを開く | ファイル選択ダイアログを開く | `Ctrl + O` | `Cmd + O` |
| 保存 | 変換結果を保存 | `Ctrl + S` | `Cmd + S` |
| 変換 | 画像をICOに変換 | `Ctrl + Enter` | `Cmd + Enter` |
| 設定 | 設定画面を開く | `Ctrl + ,` | `Cmd + ,` |
| ヘルプ | ヘルプを表示 | `F1` | `F1` |
| 終了 | アプリケーションを終了 | `Alt + F4` | `Cmd + Q` |

#### ナビゲーション

| キー | 機能 |
|------|------|
| `Tab` | 次の要素にフォーカス移動 |
| `Shift + Tab` | 前の要素にフォーカス移動 |
| `Enter` / `Space` | ボタンやリンクを実行 |
| `Esc` | ダイアログを閉じる |
| `Arrow Keys` | リスト内を移動 |
| `Home` | 最初の要素に移動 |
| `End` | 最後の要素に移動 |

#### フォーカス管理

- **フォーカスインジケーター**: すべてのフォーカス可能な要素に明確な視覚的インジケーターを表示
- **フォーカス順序**: 論理的な順序でフォーカスが移動
- **フォーカストラップ**: モーダルダイアログ内でフォーカスが循環

### キーボード操作の例

#### 画像を変換する

1. `Ctrl + O`（macOS: `Cmd + O`）でファイル選択ダイアログを開く
2. `Arrow Keys`でファイルを選択
3. `Enter`で選択を確定
4. `Tab`で「透明度保持」オプションに移動
5. `Space`でオプションを切り替え
6. `Tab`で「ICOに変換」ボタンに移動
7. `Enter`で変換を実行

#### 設定を変更する

1. `Ctrl + ,`（macOS: `Cmd + ,`）で設定画面を開く
2. `Tab`で設定項目を移動
3. `Space`でチェックボックスを切り替え
4. `Arrow Keys`でラジオボタンを選択
5. `Enter`で変更を保存

## スクリーンリーダー対応

IconConverterは、主要なスクリーンリーダーと互換性があります。

### 対応スクリーンリーダー

| スクリーンリーダー | OS | 対応状況 |
|-------------------|-----|----------|
| NVDA | Windows | ✅ 完全対応 |
| JAWS | Windows | ✅ 完全対応 |
| Narrator | Windows | ✅ 完全対応 |
| VoiceOver | macOS | ✅ 完全対応 |
| Orca | Linux | ✅ 完全対応 |

### スクリーンリーダーの設定

#### Windows（NVDA）

1. **NVDAのインストール**
   - [NVDA公式サイト](https://www.nvaccess.org/)からダウンロード
   - インストール後、自動的に起動

2. **IconConverterでの使用**
   - IconConverterを起動
   - NVDAが自動的にUI要素を読み上げます
   - `Insert + Down Arrow`で現在のフォーカス要素を読み上げ

#### macOS（VoiceOver）

1. **VoiceOverの有効化**

   ```
   システム環境設定
   → アクセシビリティ
   → VoiceOver
   → VoiceOverを有効にする
   ```

   または、`Cmd + F5`

2. **IconConverterでの使用**
   - IconConverterを起動
   - `VO + A`（`Ctrl + Option + A`）でアプリケーションを読み上げ
   - `VO + Right Arrow`で次の要素に移動

#### Linux（Orca）

1. **Orcaのインストール**

   ```bash
   sudo apt-get install orca
   ```

2. **Orcaの起動**

   ```bash
   orca
   ```

   または、`Super + Alt + S`

3. **IconConverterでの使用**
   - IconConverterを起動
   - Orcaが自動的にUI要素を読み上げます

### ARIA属性

IconConverterは、適切なARIA属性を使用しています:

- `aria-label`: 要素の説明
- `aria-describedby`: 追加の説明
- `aria-live`: 動的コンテンツの通知
- `aria-busy`: 処理中の状態
- `role`: 要素の役割

### スクリーンリーダーでの操作例

#### 画像を変換する（NVDA）

1. IconConverterを起動
   - NVDA: 「IconConverter、ウィンドウ」
2. `Tab`でファイル選択ボタンに移動
   - NVDA: 「ファイルを選択、ボタン」
3. `Enter`でファイル選択ダイアログを開く
   - NVDA: 「ファイルを開く、ダイアログ」
4. ファイルを選択して`Enter`
   - NVDA: 「test.png、選択されました」
5. `Tab`で変換ボタンに移動
   - NVDA: 「ICOに変換、ボタン」
6. `Enter`で変換を実行
   - NVDA: 「変換中、プログレスバー、50%」
   - NVDA: 「変換が完了しました」

## 視覚的アクセシビリティ

### 色のコントラスト

IconConverterは、WCAG 2.1 AAレベルのコントラスト比を満たしています。

#### ライトモード

| 要素 | 前景色 | 背景色 | コントラスト比 | 基準 |
|------|--------|--------|----------------|------|
| 通常テキスト | #1a1a1a | #ffffff | 16.1:1 | ✅ AAA |
| ボタン | #ffffff | #2563eb | 8.6:1 | ✅ AAA |
| リンク | #2563eb | #ffffff | 8.6:1 | ✅ AAA |
| エラー | #dc2626 | #ffffff | 5.9:1 | ✅ AA |

#### ダークモード

| 要素 | 前景色 | 背景色 | コントラスト比 | 基準 |
|------|--------|--------|----------------|------|
| 通常テキスト | #f5f5f5 | #1a1a1a | 15.8:1 | ✅ AAA |
| ボタン | #ffffff | #3b82f6 | 7.2:1 | ✅ AAA |
| リンク | #60a5fa | #1a1a1a | 7.5:1 | ✅ AAA |
| エラー | #f87171 | #1a1a1a | 6.2:1 | ✅ AA |

### 高コントラストモード

Windows高コントラストモードに対応しています。

**有効化方法**:

```
設定
→ 簡単操作
→ ハイコントラスト
→ ハイコントラストをオンにする
```

IconConverterは、システムの高コントラスト設定を自動的に検出し、適切なスタイルを適用します。

### 拡大表示

#### ズーム機能

- **ズームイン**: `Ctrl + +`（macOS: `Cmd + +`）
- **ズームアウト**: `Ctrl + -`（macOS: `Cmd + -`）
- **リセット**: `Ctrl + 0`（macOS: `Cmd + 0`）
- **範囲**: 50% - 200%

#### システムの拡大鏡

**Windows**:

```
Windowsキー + +
```

**macOS**:

```
システム環境設定
→ アクセシビリティ
→ ズーム機能
```

**Linux**:

```
設定
→ ユニバーサルアクセス
→ 拡大
```

### 色覚異常への対応

IconConverterは、色だけに依存しない設計を採用しています。

- **アイコン**: 色に加えて形状で情報を伝達
- **ステータス**: 色に加えてテキストで状態を表示
- **エラー**: 赤色に加えてアイコンとテキストで通知

#### 色覚シミュレーション

開発者ツールで色覚シミュレーションを確認できます:

1. `Ctrl + Shift + I`（macOS: `Cmd + Option + I`）で開発者ツールを開く
2. `Rendering`タブを選択
3. `Emulate vision deficiencies`で色覚異常をシミュレート

### フォントとテキスト

- **フォントサイズ**: 最小14px（本文）
- **行間**: 1.5倍
- **フォントファミリー**: システムフォント（読みやすさ重視）
- **テキストの拡大**: 200%まで拡大可能（レイアウト崩れなし）

## 聴覚的アクセシビリティ

IconConverterは、音声に依存しない設計を採用しています。

### 視覚的フィードバック

すべての通知とフィードバックは、視覚的に表示されます。

- **成功**: 緑色のチェックマークとメッセージ
- **エラー**: 赤色の警告アイコンとメッセージ
- **情報**: 青色の情報アイコンとメッセージ
- **警告**: 黄色の警告アイコンとメッセージ

### 音声通知の無効化

音声通知は、デフォルトで無効になっています。

**有効化方法**（オプション）:

```
設定
→ アクセシビリティ
→ 音声通知: 有効
```

## 運動機能のアクセシビリティ

### マウス不要の操作

すべての機能は、キーボードのみで操作できます。

### ドラッグ&ドロップの代替

ドラッグ&ドロップが困難な場合、以下の代替方法があります:

1. **ファイル選択ボタン**
   - `Ctrl + O`（macOS: `Cmd + O`）
   - または、「ファイルを選択」ボタンをクリック

2. **右クリックメニュー**（ファイル関連付け有効時）
   - ファイルを右クリック
   - 「ICOに変換」を選択

### クリックターゲットのサイズ

すべてのクリック可能な要素は、WCAG 2.1の基準を満たしています。

- **最小サイズ**: 44×44ピクセル
- **推奨サイズ**: 48×48ピクセル
- **間隔**: 最低8ピクセル

### スティッキーキー

Windows、macOS、Linuxのスティッキーキー機能に対応しています。

**Windows**:

```
設定
→ 簡単操作
→ キーボード
→ 固定キー機能を使う
```

**macOS**:

```
システム環境設定
→ アクセシビリティ
→ キーボード
→ スティッキーキーを有効にする
```

**Linux**:

```
設定
→ ユニバーサルアクセス
→ 入力
→ スティッキーキー
```

## 認知的アクセシビリティ

### シンプルで明確なUI

- **一貫したレイアウト**: すべての画面で一貫したデザイン
- **明確なラベル**: すべてのボタンとフォームに明確なラベル
- **視覚的階層**: 重要な要素を強調

### エラーメッセージ

エラーメッセージは、明確で具体的です。

**悪い例**:

```
エラーが発生しました。
```

**良い例**:

```
ファイルが大きすぎます。
ファイルサイズは10MB以下にしてください。
現在のファイルサイズ: 15.2MB
```

### ヘルプとガイダンス

- **ツールチップ**: すべてのボタンとオプションにツールチップを提供
- **コンテキストヘルプ**: 各画面にヘルプボタンを配置
- **ユーザーマニュアル**: 詳細なマニュアルを提供

### タイムアウトなし

IconConverterには、タイムアウトがありません。ユーザーは自分のペースで操作できます。

### 自動再生なし

アニメーションや動画の自動再生はありません。

### アニメーションの制御

**アニメーションの無効化**:

```
設定
→ アクセシビリティ
→ アニメーションを減らす: 有効
```

または、システム設定を使用:

**Windows**:

```
設定
→ 簡単操作
→ ディスプレイ
→ Windowsでアニメーションを表示する: オフ
```

**macOS**:

```
システム環境設定
→ アクセシビリティ
→ ディスプレイ
→ 視差効果を減らす
```

**Linux**:

```
設定
→ ユニバーサルアクセス
→ 視覚効果
→ アニメーションを無効にする
```
